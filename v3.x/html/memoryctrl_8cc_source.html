00052 /* overview of MC models:
00053  * McPAT memory controllers are modeled according to large number of industrial data points.
00054  * The Basic memory controller architecture is base on the Synopsis designs
00055  * (DesignWare DDR2/DDR3-Lite memory controllers and DDR2/DDR3-Lite protocol controllers)
00056  * as in Cadence ChipEstimator Tool
00057  *
00058  * An MC has 3 parts as shown in this design. McPAT models both high performance MC
00059  * based on Niagara processor designs and curving and low power MC based on data points in
00060  * Cadence ChipEstimator Tool.
00061  *
00062  * The frontend is modeled analytically, the backend is modeled empirically according to
00063  * DDR2/DDR3-Lite protocol controllers in Cadence ChipEstimator Tool
00064  * The PHY is modeled based on
00065  * "A 100mW 9.6Gb/s Transceiver in 90nm CMOS for next-generation memory interfaces ," ISSCC 2006,
00066  * and A 14mW 6.25Gb/s Transceiver in 90nm CMOS for Serial Chip-to-Chip Communication," ISSCC 2007
00067  *
00068  * In Cadence ChipEstimator Tool there are two types of memory controllers: the full memory controllers
00069  * that includes the frontend as the DesignWare DDR2/DDR3-Lite memory controllers and the backend only
00070  * memory controllers as the DDR2/DDR3-Lite protocol controllers (except DesignWare DDR2/DDR3-Lite memory
00071  * controllers, all memory controller IP in Cadence ChipEstimator Tool are backend memory controllers such as
00072  * DDRC 1600A and DDRC 800A). Thus,to some extend the area and power difference between DesignWare
00073  * DDR2/DDR3-Lite memory controllers and DDR2/DDR3-Lite protocol controllers can be an estimation to the
00074  * frontend power and area, which is very close the analitically modeled results of the frontend for Niagara2@65nm
00075  *
00076  */
