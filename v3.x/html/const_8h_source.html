00117 //that with different compilers a floating point number like 3.0 would get represented as either
00118 //2.9999....or 3.00000001 and then the integer part of the floating point number (3.0) would
00119 //be computed differently depending on the compiler. What we are doing now is to replace
00120 //int (x) with (int) (x+EPSILON) where EPSILON is 0.5. This would fix such problems. Note that
00121 //this works only when x is an integer >= 0.

00123  * Sheng thinks this is more a solution to solve the simple truncate problem

00126  * Moreover, round is not correct in CACTI since when an extra fraction of bit/line is needed,
00127  * we need to provide a complete bit/line even the fraction is just 0.01.
