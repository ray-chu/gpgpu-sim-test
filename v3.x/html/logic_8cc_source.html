00955             /*
00956              * Instruction decoder is different from n to 2^n decoders
00957              * that are commonly used in row decoders in memory arrays.
00958              * The RISC instruction decoder is typically a very simple device.
00959              * We can decode an instruction by simply
00960              * separating the machine word into small parts using wire slices
00961              * The RISC instruction decoder can be approximate by the n to 2^n decoders,
00962              * although this approximation usually underestimate power since each decoded
00963              * instruction normally has more than 1 active signal.
00964              *
00965              * However, decoding a CISC instruction word is much more difficult
00966              * than the RISC case. A CISC decoder is typically set up as a state machine.
00967              * The machine reads the opcode field to determine
00968              * what type of instruction it is,
00969              * and where the other data values are.
00970              * The instruction word is read in piece by piece,
00971              * and decisions are made at each stage as to
00972              * how the remainder of the instruction word will be read.
00973              * (sequencer and ROM are usually needed)
00974              * An x86 decoder can be even more complex since
00975              * it involve  both decoding instructions into u-ops and
00976              * merge u-ops when doing micro-ops fusion.
00977              */
